## Aufgabe 3
### Angabe
 finde ein weiteres Skript das bei der Metasploitable VM sinnvolle Ergebnisse liefert

### Lösung 
```
nmap -p 21 --script ftp-anon <IP-der-Metasploitable>
```
in meinem Fall:
```
nmap -p 21 --script ftp-anon 192.168.96.131
```

#### Ausgabe:
```

Nmap scan report for 192.168.96.131
```

### Analyse der Nmap-Ausgabe (FTP – Port 21)

- Auf Port 21 läuft ein FTP-Dienst.
- Anonymer FTP-Zugriff ist erlaubt.
- Ein Login ohne gültige Zugangsdaten ist möglich.
- Man bekommt die Mac Adresse der Metasploitable

## Aufgabe 4
### Angabe 
Aufgabe 4: Die nmap Skripte sind in Kategorien eingeteilt, es gibt z.B. eine Kategorie vuln (Siehe Kategorie vuln). Verwende die Option --script=vuln um alle Skripte dieser Kategorie zu starten. Verwende als Ziel die Metasploitable VM. Dieser Scan dauert sehr lange, schränke daher ein auf die Ports von FTP und HTTP.

### Lösung 

FTP = Port 21     
HTTP = 80


Befehl:
```
nmap -p 21,80 --script=vuln <IP-der-Metasploitable>
```

In meinem Fall:
```
nmap -p 21,80 --script=vuln 192.168.96.131
```

#### Erklärung der Ausgabe da sie sehr lang ist
Der vuln-Scan zeigt, dass auf Port 21 ein verwundbarer FTP-Dienst läuft der die bekannte Backdoor **CVE-2011-2523** enthält und eine Root-Shell ermöglicht.  
Auf Port 80 wurden mehrere Schwachstellen im HTTP-Dienst erkannt.

#### CVE Nummer(CVE-2011-2523) und genaueres darüber

Die Schwachstelle "CVE-2011-2523" betrifft den FTP‑Server vsftpd Version 2.3.4.  
Diese Version enthält eine eingebaute Backdoor, durch die ein Angreifer ohne Authentifizierung Zugriff auf eine Root-Shell erhalten kann.  
Die Schwachstelle gilt als kritisch, da sie eine vollständige Übernahme des Systems ermöglicht.
