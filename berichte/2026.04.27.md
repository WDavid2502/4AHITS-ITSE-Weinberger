## Übung (Analyse und Symbolic Mode)
### Angabe
Rechte auslesen: Ermittle die Permissions, den User Owner und den Group Owner der Datei /tmp/lab/wrestle_plans.txt?
Private Files: Erstelle eine Datei notes.txt. Ändere die Permissions so, dass der other-Klasse (o) das Read-Recht (r) entzogen wird. Nutze den symbolischen Modus von chmod.
Group Sharing: Setze die Permissions für notes.txt so, dass die group (g) die Datei lesen (r) und schreiben (w) darf, während der user (u) alle Rechte behält und others (o) keinerlei Zugriff haben. Verwende für die Änderung einen einzigen chmod Aufruf.
Skript-Vorbereitung: Du hast eine Datei myscript.sh. Welchen Befehl nutzt du, um dem user (u) und der group (g) das Recht zu geben, dieses File auszuführen (x)?
Kollektive Änderung: Entziehe mit einem einzigen Befehl im symbolischen Modus sowohl der group (g) als auch others (o) die Write- (w) und Execute-Rechte (x) für die Datei data.txt.

### Lösung
#### Rechte
Die Rechte kann man mit folgendem Befehl auslesen: 
```
ls -l /tmp/lab/wrestle_plans.txt
```

Das Ergebnis ist: 
```
-rwxr-xr-x 1 root root 6 Apr 27 05:54
```

Permissions: -rwxr-xr-x
User Owner: root
Group Owner: root

#### Private Files
Die Permissions werden nur für die spezielle other-Klasse entzogen, dies geht mit folgendem Befehl:
```
touch notes.txt
chmod o-r notes.txt
```
Others können jetzt notes.txt nicht mehr lesen.

#### Group Sharing
Damit wie angegeben der User alles darf, die Group nur schreiben und lesen und Others gar nichts darf, muss man diesen Befehl absetzen. Das "=" verpackt das alles in einen Befehl so wie in der Angabe gefordert.

```
chmod u=rwx,g=rw,o= notes.txt
```

#### Skript-Vorbereitung
Es sollten nur User und Group das Skript ausführen dürfen, dies geht mit folgendem Befehl:
```
chmod ug+x myscript.sh
```
#### Kollektive Änderung
Damit die Schreibe- und Ausführeberechtigungen entzogen werden braucht man folgenden Befehl:
```
chmod go-wx data.txt
```
